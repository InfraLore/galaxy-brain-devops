## Abstract

What would DevOps look like if your workflows were easier to discover than your
browser history?

Modern DevOps involves dozens of tools and commands that are difficult to
remember and even harder
to find. Engineers search for the same Kubernetes, Docker, or cloud command-line
instructions while
operational knowledge becomes scattered across documentation, chat threads,
shell history, and memory.

This session introduces executable knowledge: capturing operational workflows as
commands that both
document and perform the work. Using GNU Make as an orchestration layer,
engineers can turn scattered
troubleshooting steps into discoverable workflows such as `make debug` or `make
deploy`.

These commands become living documentation that evolves with the systems they
operate. The session
includes an overview of GNU Make in a DevOps context and a live demonstration of
an AI agent
interacting safely with a project's Makefile. We promise not to burn down
production -- but we will
have some fun.

Attendees will learn practical patterns for turning operational knowledge into
discoverable workflows
that reduce cognitive load and make complex infrastructure easier to operate.
